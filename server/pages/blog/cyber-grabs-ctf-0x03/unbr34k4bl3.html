Cyber Grabs CTF 0x03 – Unbr34k4bl3

Authors

Unbr34k4bl3

No one can break my rsa encryption, prove me wrong !!

Flag Format: cybergrabs{}

Author: Mritunjya

output.txt source.py

Looking at the source code, this challenge looks like a typical RSA challenge at first, but there are some important differences to note:

  • n=pqrn = pqrn=pqr (line 34). This is a twist but RSA strategies can easily be extended to 3 prime components.
  • p,q≡3 mod 4p, q \equiv 3 \bmod 4p,q≡3mod4 (line 19). This suggests that the cryptosystem is actually a Rabin cryptosystem.
  • We’re not given the public keys e1e_1e1​ and e2e_2e2​, but they are related through xxx.

Finding e1e_1e1​ and e2e_2e2​

We know that e1e_1e1​ and e2e_2e2​ are related through xxx, which is some even number greater than 2, but we’re not given any of their real values. We’re also given through an oddly-named functor function that:

1+e1+e12+⋯+e1x=1+e2+e221 + e_1 + e_1^2 + \cdots + e_1^x = 1 + e_2 + e_2^21+e1​+e12​+⋯+e1x​=1+e2​+e22​

Taking the entire equation  mod  e1\bmod\ e_1mod e1​ gives us:

1≡1+e2+e22(mode1) 0≡e2+e22 0≡e2(1+e2)\begin{aligned} 1 &\equiv 1 + e_2 + e_2^2 \pmod {e_1} \\\ 0 &\equiv e_2 + e_2^2 \\\ 0 &\equiv e_2(1 + e_2) \end{aligned}1 0 0​≡1+e2​+e22​(mode1​)≡e2​+e22​≡e2​(1+e2​)​

This means there are two possibilities: either e1=e2e_1 = e_2e1​=e2​ or e1e_1e1​ is even (since we know e2e_2e2​ is a prime). The first case isn’t possible, because with x>2x > 2x>2, the geometric series equation would not be satisfied. So it must be true that e1=2\boxed{e_1 = 2}e1​=2​, the only even prime.

Applying geometric series expansion, 1+e2+e22=2x+1−11 + e_2 + e_2^2 = 2^{x + 1} - 11+e2​+e22​=2x+1−1. We can rearrange this via the quadratic equation to e2=−1±1−4(2−2x+1)2e_2 = \frac{-1 \pm \sqrt{1 - 4 (2 - 2^{x + 1})}}{2}e2​=2−1±1−4(2−2x+1)​​. Trying out a few values we see that only x=4\boxed{x = 4}x=4​ and e2=5\boxed{e_2 = 5}e2​=5​ gives us a value that make e2e_2e2​ prime.

Finding ppp and qqq

We’re not actually given ppp or qqq, but we are given ip=p−1 mod qip = p^{-1} \bmod qip=p−1modq and iq=q−1 mod piq = q^{-1} \bmod piq=q−1modp. In order words:

p×ip≡1 mod q q×iq≡1 mod p\begin{aligned} p \times ip &\equiv 1 \bmod q \\\ q \times iq &\equiv 1 \bmod p \end{aligned}p×ip q×iq​≡1modq≡1modp​

We can rewrite these equations without the mod by introducing variables k1k_1k1​ and k2k_2k2​ to be arbitrary constants that we solve for later:

p×ip=1+k1q q×iq=1+k2p\begin{aligned} p \times ip &= 1 + k_1q \\\ q \times iq &= 1 + k_2p \end{aligned}p×ip q×iq​=1+k1​q=1+k2​p​

We’ll be trying to use these formulas to create a quadratic that we can use to eliminate k1k_1k1​ and k2k_2k2​. Multiplying these together gives:

(p×ip)(q×iq)=(1+k1q)(1+k2p) pq×ip×iq=1+k1q+k2p+k1k2pq\begin{aligned} (p \times ip)(q \times iq) &= (1 + k_1q)(1 + k_2p) \\\ pq \times ip \times iq &= 1 + k_1q + k_2p + k_1k_2pq \end{aligned}(p×ip)(q×iq) pq×ip×iq​=(1+k1​q)(1+k2​p)=1+k1​q+k2​p+k1​k2​pq​

I grouped ppp and qqq together here because it’s important to note that since we have xxx, we know rrr and thus pq=nrpq = \frac{n}{r}pq=rn​. This means that for purposes of solving the equation, pqpqpq is a constant to us. This actually introduces an interesting structure on the right hand side, we can create 2 new variables:

α=k1q β=k2p\begin{aligned} \alpha &= k_1q \\\ \beta &= k_2p \end{aligned}α β​=k1​q=k2​p​

Substituting this into our equation above we get:

pq×ip×iq=1+α+β+αβ\begin{aligned} pq \times ip \times iq &= 1 + \alpha + \beta + \alpha\beta \end{aligned}pq×ip×iq​=1+α+β+αβ​

Recall from whatever algebra class you last took that (x−x0)(x−x1)=x2−(x0+x1)x+x0x1(x - x_0)(x - x_1) = x^2 - (x_0 + x_1)x + x_0x_1(x−x0​)(x−x1​)=x2−(x0​+x1​)x+x0​x1​. Since we have both αβ\alpha\betaαβ and (α+β)(\alpha + \beta)(α+β) in our equation, we can try to look for a way to isolate them in order to create our goal.

pq×ip×iq=1+k1q+k2p+k1k2pq k1k2pq=pq×ip×iq−1−k1q−k2p k1k2=ip×iq−1pq−k1p−k2q\begin{aligned} pq \times ip \times iq &= 1 + k_1q + k_2p + k_1k_2pq \\\ k_1k_2pq &= pq \times ip \times iq - 1 - k_1q - k_2p \\\ k_1k_2 &= ip \times iq - \frac{1}{pq} - \frac{k_1}{p} - \frac{k_2}{q} \end{aligned}pq×ip×iq k1​k2​pq k1​k2​​=1+k1​q+k2​p+k1​k2​pq=pq×ip×iq−1−k1​q−k2​p=ip×iq−pq1​−pk1​​−qk2​​​

1pq\frac{1}{pq}pq1​ is basically 000, and since k1k_1k1​ and k2k_2k2​ are both smaller than ppp or qqq, then we’ll approximate this using k1k2=ip×iq−1k_1k_2 = ip \times iq - 1k1​k2​=ip×iq−1. Now that k1k2k_1k_2k1​k2​ has become a constant, we can create the coefficients we need:

α+β=pq×ip×iq−1−k1k2pq αβ=k1k2pq\begin{aligned} \alpha + \beta &= pq \times ip \times iq - 1 - k_1k_2pq \\\ \alpha\beta &= k_1k_2pq \end{aligned}α+β αβ​=pq×ip×iq−1−k1​k2​pq=k1​k2​pq​
(x−α)(x−β)=0 x2−(α+β)x+αβ=0 x=(α+β)±(α+β)2−4αβ2\begin{aligned} (x - \alpha)(x - \beta) &= 0 \\\ x^2 - (\alpha + \beta)x + \alpha\beta &= 0 \\\ x &= \frac{(\alpha+\beta) \pm \sqrt{(\alpha+\beta)^2 - 4\alpha\beta}}{2} \end{aligned}(x−α)(x−β) x2−(α+β)x+αβ x​=0=0=2(α+β)±(α+β)2−4αβ​​​

Putting this into Python, looks like:

from decimal import Decimal
getcontext().prec = 3000 # To get all digits

k1k2 = ip * iq - 1
alpha_times_beta = k1k2 * pq
alpha_plus_beta = pq * ip * iq - 1 - k1k2 * pq

def quadratic(b, c):
  b, c = Decimal(b), Decimal(c)
  disc = b ** 2 - 4 * c
  return (-b + disc.sqrt()) / 2, (-b - disc.sqrt()) / 2

alpha, beta = quadratic(-alpha_plus_beta, alpha_times_beta)

Now that we have α\alphaα and β\betaβ, we can try GCD’ing them against pqpqpq to get ppp and qqq:

from math import gcd

p = gcd(pq, int(alpha))
q = gcd(pq, int(beta))
assert p * q == pq # Success!

Alternative method

@sahuang used the sympy library to do this part instead, resulting in much less manual math. It’s based on this proof from Math StackExchange that p⋅(p−1 mod q)+q⋅(q−1 mod p)=pq+1p \cdot (p^{-1} \bmod q) + q \cdot (q^{-1} \bmod p) = pq + 1p⋅(p−1modq)+q⋅(q−1modp)=pq+1.

from sympy import *
p,q = symbols("p q")
eq1 = Eq(ip * p + iq * q - pq - 1, 0)
eq2 = Eq(p * q, pq)
sol = solve((eq1, eq2), (p, q))

Decrypting the ciphertexts

Now that we know ppp and qqq, it’s time to plug them back into the cryptosystem and get our plaintexts. c2c_2c2​ is actually easier than c1c_1c1​, because with e2=5e_2 = 5e2​=5 we can just find the modular inverse:

phi = (p - 1) * (q - 1) * (r - 1)
d2 = pow(e2, -1, phi)
m2 = pow(c2, d2, n)
print(long_to_bytes(m2))
# ... The last part of the flag is: 8ut_num83r_sy5t3m_15_3v3n_m0r3_1nt3r35t1n6} ...

This trick won’t work with c1c_1c1​ however:

d1 = pow(e1, -1, phi)
# ValueError: base is not invertible for the given modulus

Because ϕ\phiϕ is even (it’s the product of one less than 3 primes), there can’t possibly be a d1d_1d1​ such that 2⋅d1≡1 mod ϕ2 \cdot d_1 \equiv 1 \bmod \phi2⋅d1​≡1modϕ. According to Wikipedia, the decryption for a standard two-prime nnn takes 3 steps:

  1. Compute the square root of c mod pc \bmod pcmodp and c mod qc \bmod qcmodq:
    • mp=c14(p+1) mod pm_p = c^{\frac{1}{4}(p + 1)} \bmod pmp​=c41​(p+1)modp
    • mq=c14(q+1) mod qm_q = c^{\frac{1}{4}(q + 1)} \bmod qmq​=c41​(q+1)modq
  2. Use the extended Euclidean algorithm to find ypy_pyp​ and yqy_qyq​ such that yp⋅p+yq⋅q=1y_p \cdot p + y_q \cdot q = 1yp​⋅p+yq​⋅q=1.
  3. Use the Chinese remainder theorem to find the roots of ccc modulo nnn:
    • r1=(yp⋅p⋅mq+yq⋅q⋅mp) mod nr_1 = (y_p \cdot p \cdot m_q + y_q \cdot q \cdot m_p) \bmod nr1​=(yp​⋅p⋅mq​+yq​⋅q⋅mp​)modn
    • r2=n−r1r_2 = n - r_1r2​=n−r1​
    • r3=(yp⋅p⋅mq−yq⋅q⋅mp) mod nr_3 = (y_p \cdot p \cdot m_q - y_q \cdot q \cdot m_p) \bmod nr3​=(yp​⋅p⋅mq​−yq​⋅q⋅mp​)modn
    • r4=n−r3r_4 = n - r_3r4​=n−r3​
  4. The real message could be any rir_iri​, but we don’t know which.

Converting this to work with n=pqrn = pqrn=pqr, it looks like:

  1. Compute the square root of c mod pc \bmod pcmodp, c mod qc \bmod qcmodq, and c mod rc \bmod rcmodr:
    • mp=c14(p+1) mod pm_p = c^{\frac{1}{4}(p + 1)} \bmod pmp​=c41​(p+1)modp
    • mq=c14(q+1) mod qm_q = c^{\frac{1}{4}(q + 1)} \bmod qmq​=c41​(q+1)modq
    • mr=c14(r+1) mod rm_r = c^{\frac{1}{4}(r + 1)} \bmod rmr​=c41​(r+1)modr
  2. Using the variable names from AoPS’s definition of CRT:
    • For k∈{p,q,r},bk=nkk \in \{ p, q, r \}, b_k = \frac{n}{k}k∈{p,q,r},bk​=kn​.
    • For k∈{p,q,r},ak⋅bk≡1 mod kk \in \{ p, q, r \}, a_k \cdot b_k \equiv 1 \bmod kk∈{p,q,r},ak​⋅bk​≡1modk.
  3. Let r=∑k{p,q,r}±(ak⋅bk⋅mk) mod nr = \displaystyle\sum_k^{\{ p, q, r \}} \pm (a_k \cdot b_k \cdot m_k) \bmod nr=k∑{p,q,r}​±(ak​⋅bk​⋅mk​)modn.
  4. The real message could be any rrr, but we don’t know which.

In code this looks like:

# Step 1
mp = pow(c1, (p + 1) // 4, p)
mq = pow(c1, (q + 1) // 4, q)
mr = pow(c1, (r + 1) // 4, r)

# Step 2
bp = n // p
bq = n // q
br = n // r
ap = pow(bp, -1, p)
aq = pow(bq, -1, q)
ar = pow(br, -1, r)

# Step 3
from itertools import product
for sp, sq, sr in product((-1, 1), repeat=3):
  m = (sp * ap * bp * mp + sq * aq * bq * mq + sr * ar * br * mr) % n
  m = long_to_bytes(m)

  # Step 4
  # We know that the real flag starts with `cybergrabs{`...
  if b"cybergrabs" in m: print(m)

# Congratulations, You found the first part of flag cybergrabs{r481n_cryp70sy5t3m_15_1nt3r35t1n6_ ...

The final flag, then, is:

cybergrabs{r481n_cryp70sy5t3m_15_1nt3r35t1n6_8ut_num83r_sy5t3m_15_3v3n_m0r3_1nt3r35t1n6}
